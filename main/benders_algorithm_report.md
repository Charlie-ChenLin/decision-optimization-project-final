## 1. 算法设计：Benders 分解算法

由于两阶段随机规划模型在场景数 $S$ 增加时，第二阶段变量和约束的数量呈爆炸式增长，直接使用 Gurobi 求解面临内存溢出和求解时间过长的性能瓶颈。因此，我们采用 **Benders 分解算法（L-shaped Method）** 将原问题分解为：
- **主问题 (Master Problem)**：处理第一阶段与场景无关的投资决策（如设备购买、区域电网改造等），并用辅助变量 $\theta_s$ 预估第二阶段成本。
- **子问题 (Subproblem)**：在第一阶段决策固定的情况下，针对每个场景 $s$ 独立求解运营与碳排放成本。

### 算法伪代码 (可直接放入 LaTeX/PPT)

```text
Algorithm 1: Benders Decomposition (L-shaped Method) for Port Electrification
--------------------------------------------------------------------------------
Input: 容忍度 tol, 最大迭代次数 max_iter, 场景集合 S
Output: 第一阶段最优决策 (x2, y, a, b2), 最优目标值

1: 初始化 LB = 0, UB = +∞, iter = 0
2: 初始化主问题 (Master Problem)，仅包含第一阶段变量和约束，以及预估成本变量 \theta_s = 0
3: while iter < max_iter do
4:     求解主问题，得到第一阶段决策解 (\bar{x2}, \bar{y}, \bar{a}) 及 \bar{\theta_s}
5:     更新下界 LB = Master_ObjVal
6:     当前二阶段实际总成本 current_sp_total = 第一阶段固定成本
7:     
8:     for s in S do
9:         将 (\bar{x2}, \bar{y}) 作为已知常量代入场景 s 的子问题
10:        求解子问题 SP_s
11:        if SP_s 存在最优解 then
12:            current_sp_total += SP_s_ObjVal / |S|
13:            if \bar{\theta_s} < SP_s_ObjVal then
14:                获取对偶变量 (Pi)，向主问题添加 最优性割 (Optimality Cut)
15:        else if SP_s 不可行 then
16:            获取极射线 (FarkasDual)，向主问题添加 可行性割 (Feasibility Cut)
17:            current_sp_total = +∞
18:            break
19:        end if
20:    end for
21:
22:    更新上界 UB = min(UB, current_sp_total)
23:    if (UB - LB) / LB < tol then
24:        满足收敛条件，跳出循环
25:    end if
26:    iter = iter + 1
27: end while
28: return 最优解与目标值
```

## 2. 算法性能瓶颈分析与实验结论

基于 `run_algo_comparison.py` 的实验结果，我们可以得出以下关于算法绕过 Gurobi 性能瓶颈的文字说明（可写入报告“数值实验”章节）：

**直接求解的瓶颈**：
在多场景随机规划中，Gurobi 直接求解需要将所有场景的变量（如 $w_{szt}, x_{szt}$）放入同一个庞大的约束矩阵中。在我们的实验中，随着规模从“小规模 (5场景)”增加到“大规模 (30场景)”，约束数量急剧上升。虽然商业求解器在处理几万个变量的稀疏矩阵时表现优秀（小规模仅需0.05秒），但当场景数扩展到几百甚至上千（真实环境）时，内存消耗将呈指数级增长，最终导致 Gurobi 无法初始化模型（Out of Memory）。

**Benders 分解的优势与逻辑**：
1. **空间解耦**：Benders 分解通过主从结构，每次只将**单个场景**的约束加载到内存中独立求解。在上述实验中，无论是 5 个场景还是 30 个场景，子问题的矩阵规模始终是恒定的，极大降低了内存开销。
2. **多割生成加速 (Multi-cut)**：我们在算法中为每个场景单独定义了 $\theta_s$，在每次迭代中可以同时向主问题添加多个最优性割（Multi-cut）。这使得主问题能够更快地逼近第二阶段的真实成本面。实验结果显示，中规模在 23 次迭代后即可收敛（Gap < 0.05%），证明了割平面的高效性。
## 3. 启发式算法探索（加分项）：均值指导启发式 (Mean-Value Guided Heuristic)

在作业要求中提到鼓励探索多种求解算法（例如：场景分解、渐进启发式等）。为了在极短的时间内获得大规模随机规划问题的高质量可行解，我们设计并实现了一种“均值指导启发式”算法。

**算法思路：**
两阶段随机规划问题中最难求解的部分是：第一阶段的**离散（整数）决策**（例如：某个区域在何时进行电网改造变量 $a_{zt}$）会与第二阶段海量场景下的连续决策产生复杂的组合爆炸。
1. **降维与指导**：首先，我们将所有场景取期望，构建一个“确定性均值模型 (EV)”。由于仅有 1 个场景，该模型可被极快求解（秒级）。
2. **固定离散决策**：将均值模型中求得的第一阶段最核心的整数决策（如电网改造时序 $a_{zt}$）作为固定参数代入原随机规划模型。
3. **松弛与暖启动**：一旦电网架构固定，原模型中复杂的逻辑约束被解耦，模型从一个复杂的混合整数规划 (MIP) 退化为一个极易求解的线性规划 (LP) 或简化的 MIP。我们同时将均值模型的连续变量（如购买量 $b_2$ 和改造量 $y$）作为暖启动（MIP Start）提供给求解器。

**性能表现（参考上方实验结果汇总）：**
在 `run_algo_comparison.py` 的测试中：
- **中规模 (20场景)**：启发式算法仅耗时 **0.23s**，而 Benders 需要 5.56s，且求得的目标值 (1855133) 与直接求解完全一致。
- **大规模 (30场景)**：启发式算法耗时仅 **1.53s**，极大地低于 Benders 分解的 207s 和 直接求解的时间。其目标值为 3414557，与最优解 (3392050) 相比，**误差仅为 0.6%**。

**结论**：
均值指导启发式算法能够在牺牲不到 1% 解的精度的情况下，将大规模问题的求解时间缩短两个数量级（从数分钟降至 1-2 秒）。这种方法非常适合在真实港口运营中，当决策时间极其受限（如突发事件需要快速重排计划）时使用，体现了理论与工程实践的有效结合。
