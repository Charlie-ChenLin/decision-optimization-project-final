# 列生成/场景生成式算法报告

## 算法定位

本算法对应作业要求中提到的“列生成”思想。严格的列生成通常在主问题中逐步加入变量列；在两阶段随机规划场景下，可以采用与其等价的“场景生成/约束生成”思路：不一次性纳入所有需求场景，而是从少量代表性场景开始求解，再逐步加入高压力场景，形成小规模主问题的序列。

## 核心思想

1. 对每个场景计算压力分数，压力分数综合考虑该场景的总需求量和相对峰值需求。
2. 初始只选择若干高压力场景，构建缩减版两阶段随机规划并求解。
3. 将缩减模型得到的第一阶段决策固定，回到全场景模型中评估真实期望成本。
4. 每轮继续加入更多高压力场景，重复求解和评估。
5. 在所有迭代中保留全场景评估成本最低的第一阶段方案。

## 伪代码

```text
Input: 场景集合 S, 初始场景数 n0, 每轮加入场景数 q, 最大迭代次数 K
Output: 第一阶段决策和全场景评估成本

1. 根据场景总需求与峰值需求计算 score(s)
2. 按 score(s) 从高到低排序
3. selected ← 排名前 n0 的场景
4. best_obj ← +∞
5. for k = 1,...,K do
6.     用 selected 构建缩减随机规划模型
7.     求解缩减模型，得到第一阶段决策 x_k
8.     固定 x_k，在全场景模型中评估期望成本 obj_k
9.     if obj_k < best_obj then
10.        best_obj ← obj_k, best_solution ← x_k
11.    end if
12.    selected ← selected ∪ 下 q 个高压力场景
13. end for
14. return best_solution, best_obj
```

## 优势与局限

优势：
- 不需要一开始构建完整大规模模型，适合场景数量很大时快速筛选决策。
- 高压力场景优先进入模型，能较快获得鲁棒性较强的投资方案。
- 可作为 Gurobi 完整求解前的 warm start 或作为启发式近似方案。

局限：
- 该算法本质是启发式场景生成，不保证全局最优。
- 如果低需求场景对投资成本权衡很重要，仅选择高压力场景可能导致投资偏保守。
